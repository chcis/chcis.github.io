# java spring用注解还是xml？

学完spring，接触到了框架，总算踏进了java的大门。

  • 起初疑惑到底用spring注解好还是xml好？在这里spring官方也给出了建议：

我们可以了解到：

  • 基于xml配置spring的优点是方便集中管理，但是代码冗长，这也是很多程序员讨厌java的一点，毕竟在其他技术上xml已经落后，有更好的替代了。

  • 基于注解配置spring的特点是简短简洁，但是要嵌入到各个文件中，不方便改善管理。

  • 其实，不管是xml还是注解方式，都是实现IOC的一种辅助方式，并没有违背IOC的初衷。

总结：

简单的依赖注入建议使用注解，复杂的建议使用xml，可以混合使用。

但是混合使用要注意，注解的注入是在XML之前执行的，所以如果同时使用注解和XML注入同一个属性，XML会覆盖掉注解的注入。
